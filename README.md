E-Commerce Order Management System – Requirement Analysis and Software Requirement Specification (Week 1)

## Project Description
This project focuses on analyzing the business requirements of an E-Commerce Order Management System and preparing a Software Requirement Specification (SRS) document. The analysis serves as the foundation for designing the database in the upcoming weeks.

The system manages customer registration, product catalog, inventory, orders, payments, shipments, and customer reviews using a centralized database.

---

## Project Objectives

- Analyze the business requirements of the proposed system.
- Identify the core entities and their purpose.
- Identify stakeholders and their responsibilities.
- Define functional and non-functional requirements.
- Specify the project scope.
- Identify assumptions and constraints.
- Prepare a complete Software Requirement Specification (SRS).

---

## Business Scenario

A rapidly growing e-commerce company requires a centralized database system to manage daily business operations efficiently. The system should support:

- Customer Registration
- Product Management
- Inventory Tracking
- Order Processing
- Payment Management
- Shipment Tracking
- Customer Reviews
- Business Report Generation

The database should improve data accuracy, reduce redundancy, and support efficient business operations.

---

## Core Entities

The project uses the following mandatory entities:

1. Customer

3. Product
4. Supplier
5. Order
6. Order Details
7. Payment
